# 代码风格
- 启动脚本需要能直接在项目根目录下通过 python xxx.py 运行，因此要注意 import 的相对路径写法，确保从根目录执行时不会出现模块导入错误
- 你必须使用尽可能简洁和少量的代码逻辑实现功能，代码可读性高，尽量减少用户的阅读负担（但是函数签名和关键逻辑的注释一定要有）
- 测试脚本中绝不生成mock的东西，除非用户要求
- 代码不允许设计fallback机制，除非有必要，不然风险应及时暴露而不是兜底
- 修改代码用patch修改，即使是只读沙箱

# 执行风格
- 能在命令行直接用 Python 验证的测试，不要新建本地测试脚本；一次性、无沉淀价值的中间测试跑完即删，不留文件；另外在用户没主动要求情况下不要主动写README.md这种文档
- 在遇到自己不理解或者质疑的设计时，大胆向用户提问
- 用中文回答用户问题
